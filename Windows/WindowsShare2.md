# Windows 文件共享快速教程 + 常见问题解答（FAQ）

本教程适合：
- Windows 10 / Windows 11
- 家庭或办公室局域网
- 需要“输入账号密码才能访问”的共享方式（不做匿名开放）

---

## 一、5-10 分钟快速教程

## 0. 目标

1. 电脑 A 分享一个文件夹给电脑 B。  
2. B 访问时必须输入凭据（账号密码）。  
3. 你可以随时收回某个人的访问权限。  

---

## 1. 网络基础设置（A、B 都做）

1. 把当前网络改为 `专用网络`（Private）。  
2. 打开：`控制面板 -> 网络和共享中心 -> 更改高级共享设置`。  
3. 在“专用”下启用：
   - `网络发现`
   - `文件和打印机共享`
4. 在“所有网络”下启用：
   - `密码保护共享`（必须开）

---

## 2. 创建共享账号（重点：多版本多方式）

你说的情况非常常见：  
有些系统（精简版/盗版改版/策略受限的专业版）在“设置”里看不到或无法新建用户。  
因此这里给你按“可用性”分 4 种方式，能用哪种就用哪种。

### 2.1 方式 A：通过“设置”创建（最直观）

适用：
- 大部分正常安装的 Win10/Win11 家庭版/专业版

路径：
- `设置 -> 账户 -> 家庭和其他用户 -> 将其他人添加到这台电脑`

建议：
- 创建本地用户（例如 `share_user`）
- 设置强密码
- 保持标准用户（不要管理员）

如果这里没有入口、按钮灰掉、点击无反应，直接用方式 B 或 C。

### 2.2 方式 B：通过“计算机管理”创建（专业版/企业版常用）

适用：
- Win10/11 专业版、企业版、教育版
- “设置”不可用但系统管理工具正常

路径：
1. `Win + X -> 计算机管理`
2. `本地用户和组 -> 用户`
3. 右侧空白处右键 `新用户`
4. 创建 `share_user` 并设置密码（谨慎选择`密码永不过期`和`用户下次登录时须更改密码`选项）

注意：
- 很多 Home 版默认没有“本地用户和组（lusrmgr）”。
- 部分精简版/盗版系统可能也被移除。

### 2.3 方式 C：命令行创建（最稳，强烈建议掌握）

适用：
- 几乎所有版本
- 设置入口消失、管理工具被精简、远程排障场景

用管理员权限打开 `CMD` 或 `PowerShell`，执行：

```bat
net user share_user YourStrongPassword123! /add


:: 创建用户（必须有密码，空密码不建议，会影响网络共享认证）
net user 共享访客1 "P@ssw0rd123!" /add

:: （可选）加入管理员组（不建议给共享访问者管理员权限）
net localgroup Administrators 共享访客1 /add

:: （可选）确保是普通用户（默认就是 Users 组成员）
net localgroup Users 共享访客1 /add


```

给普通权限（默认已是普通用户），无需加管理员组。  
如需禁用账号：

```bat
net user share_user /active:no
```

修改密码：

```bat
net user share_user NewStrongPassword456!
```

### 2.4 方式 D：PowerShell 创建（脚本化）


适用：
- 你要批量创建或自动化部署

1. 方式1
    ```powershell
    $pwd = Read-Host "Input password" -AsSecureString
    New-LocalUser -Name "share_user" -Password $pwd -FullName "Share User"
    ```

2. 方式2

    - 以管理员打开 PowerShell：
        ```powershell
        $password = Read-Host "输入密码" -AsSecureString
        New-LocalUser -Name "共享访客1" -Password $password -FullName "Share Visitor" -Description "用于访问共享文件夹"
        Add-LocalGroupMember -Group "Users" -Member "共享访客1"
        ```

    - （可选）设为密码永不过期：
    
        ```powershell 
        Set-LocalUser -Name "共享访客1" -PasswordNeverExpires $true
        ```


    - （可选）加管理员：
        ```powershell
        Add-LocalGroupMember -Group "Administrators" -Member "共享访客1"
        ```

如果报命令不存在，改用方式 C（`net user`）。

### 2.5 快速选择规则（按这个走）

1. 先试方式 A（设置）。  
2. A 不行就试方式 B（计算机管理）。  
3. B 也没有就直接方式 C（`net user`，最稳）。  

---

## 3. 在电脑 A 创建共享目录并授权

示例目录：`D:\Share`，共享名：`Share`

### 3.1 共享权限

1. 右键 `D:\Share` -> `属性` -> `共享` -> `高级共享`。  
2. 勾选 `共享此文件夹`，共享名填 `Share`。  
3. 点 `权限`：
   - 删除 `Everyone`（推荐）
   - 添加 `share_user`
   - 按需给 `读取` 或 `更改`

### 3.2 NTFS 安全权限

1. 同目录 -> `属性` -> `安全` -> `编辑`。  
2. 添加 `share_user`。  
3. 按需给 `读取和执行` 或 `修改`。  

记住：**最终权限 = 共享权限 与 NTFS 权限的交集（更严格者生效）**。

---

## 4. 防火墙检查（A、B 建议都检查）

1. 打开 `Windows Defender 防火墙`。  
2. 在“允许应用通过防火墙”里确认以下项目在“专用网络”勾选：
   - `网络发现`
   - `文件和打印机共享`

---

## 5. 在电脑 B 访问共享

### 5.1 先拿到 A 的地址

在 A 上执行：

```powershell
ipconfig
hostname
```

记下 A 的 IPv4（例如 `192.168.1.20`）和主机名（例如 `MyPC`）。

### 5.2 访问共享

在 B 的资源管理器地址栏输入：

```text
\\192.168.1.20\Share
```

输入凭据：
- 用户名：`MyPC\share_user`
- 密码：该账号密码

### 5.3 映射网络驱动器（可选）

```powershell
net use Z: \\192.168.1.20\Share /user:MyPC\share_user * /persistent:yes
```

---

## 6. 可选：给不同人不同账号

如果你要多人访问，不要共用一个账号。  
做法：一人一个本地账号（如 `u_alice`、`u_bob`），然后分别分配权限。  
以后要禁用某人，直接改密码或禁用该用户即可。

## 二、常见问题解答（FAQ）

## Q1：Win11“网络”里看不到别的电脑，是否共享失败？

不一定。  
先直接试：`\\对方IP\共享名`。  
“网络”列表依赖网络发现服务，常见是显示层问题，不代表 SMB 不能用。

---

## Q2：专业版/精简版/盗版系统里，“设置”无法创建新用户怎么办？

按这个顺序排：
1. 先试 `Win + X -> 计算机管理 -> 本地用户和组 -> 用户`。  
2. 如果没有“本地用户和组”，直接用管理员命令行：  

```bat
net user share_user YourStrongPassword123! /add
```

3. 再去共享权限和 NTFS 权限里添加这个账号。  

结论：**“设置”不能创建用户，不影响你做凭据共享。`net user` 基本都能兜底。**

---

## Q3：提示“拒绝访问”，怎么查？

按顺序查：
1. 共享权限里是否有该账号。  
2. NTFS 安全权限里是否也有该账号。  
3. 是否启用了密码保护共享。  
4. 用户名格式是否正确（`主机名\用户名`）。  

---

## Q4：一直弹凭据窗口，输对了也进不去？

常见原因：
1. 输入成了微软账号格式或域账号，和你授权账号不一致。  
2. 客户端缓存了旧凭据。  

处理：

```powershell
net use * /delete /y
```

然后重新访问并输入 `主机名\用户名`。

---

## Q5：报错“系统错误 1219（多重连接冲突）”？

同一台服务器不能同时用两组凭据连接。  
先断开所有映射/连接（上面的 `net use * /delete /y`），再用目标账号重连。

---

## Q6：能打开但不能新建/删除文件？

你只拿到了读权限。  
需要在“共享权限”和“NTFS 权限”两边都给 `修改`，缺一不可。

---

## Q7：`ping` 通，但 `\\IP\Share` 打不开？

优先查：
1. A 上防火墙是否放行文件共享。  
2. A 上是否开启了文件和打印机共享。  
3. 共享名是否正确（区分拼写）。  
4. A 是否睡眠/关机。  

---

## Q8：局域网内能访问，外网（异地）不能访问？

这是正常现象。  
SMB 默认是局域网方案。外网访问建议先连 VPN（Tailscale/WireGuard/ZeroTier）再走 `\\VPN_IP\Share`。  
不建议直接暴露 445 端口到公网。

---

## Q9：我的设备是 NAS，步骤一样吗？

大方向一样：开启 SMB、创建用户、分配共享权限。  
区别在于账号和权限在 NAS 管理界面配置，Windows 客户端访问方式仍是：

```text
\\NAS_IP\共享名
```

---

## 三、快速排障清单（1 分钟）

1. A、B 是否同网段（如都在 `192.168.1.x`）。  
2. 网络类型是否是“专用”。  
3. 网络发现和文件共享是否已启用。  
4. 密码保护共享是否开启。  
5. 共享权限 + NTFS 权限是否都授予同一个账号。  
6. 客户端是否清理过旧凭据（`net use * /delete /y`）。  
7. 直接用 `\\IP\Share` 测试，不依赖“网络”列表。  

---

## 四、安全建议（强烈建议）

1. 不要共享整个系统盘（如 `C:\`）。  
2. 不要长期使用管理员账号做共享凭据。  
3. 默认先给只读，确实需要再开写入。  
4. 定期更改共享账号密码。  
5. 重要文件启用备份，避免误删不可恢复。
